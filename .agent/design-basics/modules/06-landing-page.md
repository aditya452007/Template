# Landing Page Design — Conversion Ka Science

> Source: Figma "16 Landing Page Examples", "29 AI Website Examples"

## Landing Page Anatomy

```
NAVBAR (Logo | Nav Links | CTA Button)
────────────────────────────────────────
HERO
├── Headline (single clear promise)
├── Subheadline (supporting text)
├── Primary CTA button
└── Hero visual (image/video/3D)
────────────────────────────────────────
SOCIAL PROOF
├── Customer logos
├── Testimonials / quotes
└── Stats / numbers
────────────────────────────────────────
FEATURES / BENEFITS
├── Feature 1 (icon + title + desc)
├── Feature 2
└── Feature 3
────────────────────────────────────────
HOW IT WORKS (3-step process)
├── Step 1 → Step 2 → Step 3
────────────────────────────────────────
FINAL CTA
├── Strong headline
├── Supporting text
└── CTA button (same or stronger)
────────────────────────────────────────
FOOTER (links, social, legal)
```

## Rules for Each Industry (Your Startup)

### SaaS / Tech Landing
- **Headline** = single biggest problem your startup solves
- **Visual** = product screenshot / dashboard / 3D mockup
- **Trust signals** = customer logos, security badges, "used by X teams"
- **CTA** = "Start free trial" / "Get early access"
- **Free tier** mention prominently

### AI Product Landing
- Don't just show a prompt box — show results
- Before/after comparisons
- Speed/performance stats
- "Built with AI" is expected — show HOW it helps

### Pricing Page (see 07-pricing-page.md for detailed)
- Separate landing page from pricing page
- Link from nav and hero CTA

## Hero Section Rules

- **Single headline** (max 10 words) — clear value prop
- **Subheadline** (max 20 words) — expands on headline
- **Single CTA** — primary action only (don't confuse)
- **Hero image/video** — reinforces message, not just decoration
- **No navigation links in hero** — keep focus on CTA

## CTA Best Practices

- High-contrast color (stands out from page)
- Large tap target (min 44px height)
- Action-oriented text ("Get Started Free" not "Submit")
- Sticky on mobile scroll
- Max 1 primary CTA per viewport
- Secondary CTA allowed but visually distinct

## Social Proof Placement

| Location | Type | Effect |
|----------|------|--------|
| Below hero | Logo bar | Builds instant trust |
| Near CTA | Testimonial | Reduces last-minute doubt |
| Sidebar | Case study link | For skeptical visitors |
| Footer | Stats/awards | Final credibility boost |

## Common Landing Page Mistakes

1. ❌ Too many CTAs — user ko confusion hoti hai
2. ❌ Weak headline — value prop clear nahi
3. ❌ Generic stock photos — brand identity weak
4. ❌ No social proof — visitors trust nahi karte
5. ❌ Mobile not optimized — 51.4% spending mobile se
6. ❌ Information overload — keep it focused
7. ❌ Slow load time — 1s vs 5s = 2.5x conversion diff
