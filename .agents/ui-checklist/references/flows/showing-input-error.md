# Showing Input Error Flow Checklist

- [ ] **Keep the input in default state** — Check for errors only after information is entered.
- [ ] **Allow user to enter information** — No interruptions while typing, don't assess mid-input.
- [ ] **Signal error after loss of focus** — Explain why the error happened and what's required. Combine text with error icon.
- [ ] **Return to default state upon reattempt** — Error message disappears on focus. Process repeats if error persists.
