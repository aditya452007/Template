# Toast Checklist

- [ ] **Copy** — Clear and concise, focusing on status or action.
- [ ] **Placement** — Corners of the viewport, not as the focus.
- [ ] **Usage** — Triggered after an action or event.
- [ ] **Variants** — Colour-coded (green for success, red for error). Don't rely solely on colour; use icons or copy.
- [ ] **Length of appearance** — Long enough to read, short enough to not obstruct. Read aloud to gauge.
- [ ] **Dismissable** — User can close; should fade away shortly if not manually dismissed.
